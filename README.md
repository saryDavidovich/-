# מערכת ניוזלטרים אוטומטית

מערכת מלאה: רשימות תפוצה לפי נושא, שו"ת דו-כיווני במייל, לוח מודעות (חינם +
מודגשת + פרימיום עם תמונה/גיף וצבעים מותאמים), תוכן חופשי ("נושאים"), ופאנל
ניהול עם תפריט צד. **כל פעולה שהלקוח עושה - שאלה, מודעה, הצטרפות, הסרה,
תגובה - ניתנת לביצוע דרך מייל בלבד**, בלי צורך בדפדפן בכלל.

**אין שום תלות ב-AI/LLM בשום מקום במערכת - אפס עלות מהצד הזה.**

## מה עובד (הכל נבדק בפועל)

### רשימות ותוכן
- יצירת נושא/רשימה חדשה דרך הפאנל
- שלושה סוגי תוכן: **שאלה+תשובה**, **מודעה** (3 רמות), **נושא/מאמר** חופשי
- כתיבה ישירה של כל סוג תוכן מהפאנל (בלי לחכות למייל מלקוח)
- תור אישור עם עריכה קלה, כולל תגובות המשך לשאלות שכבר נשלחו בעבר
- תצוגה מקדימה חיה - מראה בדיוק מה יישלח, בלחיצה אחת
- הגדרות לכל רשימה: סדר הסקשנים בגיליון, אילו כפתורים מוצגים
- היסטוריה - כל גיליון שנשלח נשמר לצמיתות ואפשר לצפות בו שוב
- ארכיון ציבורי (`/archive/[slug]`) - הלקוחות עצמם יכולים לעיין בגיליונות עבר

### הכל דרך מייל (plus-addressing על תת-הדומיין לקליטה)
| פעולה | כתובת | תוצאה |
|---|---|---|
| שאלה חדשה | `ask+[slug]@מייל-הקליטה` | נכנסת לתור אישור |
| מודעת שורה (חינם) | `ads+[slug]@...` | נכנסת לתור, רמה=חינם |
| מודעה מודגשת | `adsplus+[slug]@...` | נכנסת לתור, רמה=מודגשת |
| מודעה פרימיום | `adspremium+[slug]@...` | נכנסת לתור, רמה=פרימיום |
| נושא/מאמר | `topic+[slug]@...` | נכנס לתור |
| תגובה לשאלה | `reply+[מספר]@...` | תגובה, גם אם השאלה כבר נשלחה בעבר |
| הצטרפות | `join+[slug]@...` | נרשם מייד, לפי כתובת השולח |
| הסרה | `leave+[slug]@...` | מוסר מייד, לפי כתובת השולח |

כל הכפתורים בתוך הגיליון שנשלח בנויים כ-mailto עם הכתובת הנכונה כבר ממולאת,
כך שללקוח מספיק ללחוץ ולשלוח - גם אם יש לו רק גישה למייל (כמו "נטו מייל")
בלי דפדפן.

### תמונות במודעות
מודעות מודגשות/פרימיום יכולות לכלול תמונה/גיף (מועלה דרך הפאנל, או מצורף
במייל). **חשוב:** התמונה לא נטענת אוטומטית בגיליון - היא מוצגת כקישור טקסט
"לצפייה בתמונה" שנפתח רק בלחיצה יזומה, כדי לא לגרום לעיכוב אצל מסנני מייל
כמו נטפרי שסורקים תמונות שנטענות אוטומטית.

## מה עדיין לא כולל (ומה לעשות בהמשך)
- **תשלום בפועל** על מודעות מודגשות/פרימיום - הרמות קיימות ונשמרות, אבל
  אין עדיין חיבור סליקה (Stripe/טרנזילה). בינתיים זה עובד על בסיס אמון/מעקב ידני.
- אחסון קבצים בענן (S3/Cloudinary) - כרגע תמונות נשמרות בתיקיית
  `data/uploads` המקומית (על ה-Volume של Railway, כך שלא נמחקות בדיפלוי).

## הרצה מקומית
```bash
npm install
cp .env.example .env
# ערכו את .env: הגדירו ADMIN_PASSWORD, SESSION_SECRET, INBOUND_DOMAIN
npm start
```
פתחו http://localhost:3000/admin והתחברו עם הסיסמה שהגדרתם.

## חיבור מייל אמיתי דרך SendGrid

### שליחה
1. אמתו את הדומיין הראשי שלכם ב-SendGrid (**Settings → Sender Authentication**)
2. הגדירו ב-`.env`: `SENDGRID_API_KEY`, `FROM_ADDRESS`

### קליטה (Inbound Parse) - על תת-דומיין נפרד
חשוב: **לא** על הדומיין הראשי אם יש עליו תיבות מייל אמיתיות - צריך תת-דומיין
ייעודי (למשל `mail.yourdomain.com`):
1. ב-DNS (GoDaddy וכו'): רשומת **MX** בשם התת-דומיין, ערך `mx.sendgrid.net`, עדיפות 10
2. ב-SendGrid: **Settings → Inbound Parse → Add Host & URL**
   - Receiving Domain: תת-הדומיין
   - Destination URL: `https://הדומיין-שלכם/webhooks/inbound` (**חייב לכלול את הנתיב `/webhooks/inbound` בסוף, לא רק את השורש!**)
3. הגדירו ב-`.env`: `INBOUND_DOMAIN=mail.yourdomain.com`

## פריסה (Railway)
- Node.js/Express עם SQLite - Volume חובה על נתיב `/app/data` (או מה שנוצר
  אוטומטית דרך ה-UI), אחרת הנתונים נמחקים בכל דיפלוי
- אבחון: בלוגים, בכל עלייה, המערכת מדפיסה את הנתיב המדויק של מסד הנתונים
  ובודקת אם יש "עקבה" משימוש קודם - עוזר לוודא שה-Volume מחובר נכון

## מבנה הפרויקט
```
src/
  db.js                - סכמת מסד הנתונים + מיגרציות בטוחות לגרסאות קודמות
  templates.js          - עיצוב המייל: מותג אחיד, כפתורי mailto, תמונות ללא טעינה אוטומטית
  compiler.js            - מרכיב ושולח את הגיליון השבועי דרך SendGrid, כולל תזמון cron
  routes/
    admin.js                - פאנל ניהול: דשבורד, תור אישור, כתיבה ישירה, הגדרות, היסטוריה, תצוגה מקדימה
    inbound.js                - webhook לקליטת כל הפעולות במייל (שאלה/מודעה/נושא/תגובה/הצטרפות/הסרה)
    public.js                   - טפסים ציבוריים: מודעה, נושא, הרשמה, הסרה, ארכיון
  views/                          - תבניות EJS, עם תפריט צד משותף (partials/header.ejs + footer.ejs)
```
