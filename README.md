# מערכת ניוזלטרים אוטומטית

מערכת מלאה: רשימות תפוצה לפי נושא, שו"ת דו-כיווני במייל, לוח מודעות חינמי
(עם שכבת תשלום עתידית מוכנה אך כבויה), ותהליך אישור-ושליחה שבועי אוטומטי כמעט
לגמרי. **אין שום תלות ב-AI/LLM בשום מקום במערכת - אפס עלות מהצד הזה.**

## מה כבר עובד (נבדק בפועל)
- יצירת רשימה/נושא חדש דרך הפאנל (טופס אחד, 30 שניות).
- קליטת מייל נכנס (שאלה / מודעה) ומיון אוטומטי לרשימה הנכונה.
- תור אישור: עריכה קלה + אישור בלחיצה אחת.
- מנגנון תשובה במייל (reply-to ממוען לפי מזהה השאלה).
- הרכבת גיליון שבועי מעוצב (HTML) ושליחה לכל מנוי - נבדק end-to-end.
- מיתוג אחיד ("בית" אחד) עם גוון הדגשה שונה לכל רשימה בלבד - לא לוגו/תבנית שונים.
- הרשמה והסרה מרשימה בקישור אחד.

## מה זה עדיין לא כולל (ומה לעשות בהמשך)
- **חיבור אמיתי לספק מייל** (Mailgun מומלץ - יש שכבה חינמית). כרגע השליחה
  במצב "יבש" (dry run) שרק מדפיסה ללוג, כי אין מפתח API אמיתי מוגדר.
- **תשלום בפועל** על תמונות/גיפים/קישורים - השדות קיימים במסד הנתונים
  ובטופס (`PAID_FEATURES_ENABLED=true`), אבל חיבור סליקה (Stripe/Tranzila/כרטיס
  אשראי ישראלי) צריך להתווסף כשתהיה מוכן.
- אחסון קבצים (תמונות/גיפים) בפועל - כרגע נתמכת רק שליפת קישורי תמונה שמופיעים
  בגוף המייל; להעלאת קבצים מצורפים אמיתיים צריך לחבר S3/Cloudinary.

## הרצה מקומית
```bash
npm install
cp .env.example .env
# ערכו את .env: הגדירו ADMIN_PASSWORD, SESSION_SECRET
npm start
```
פתחו http://localhost:3000/admin והתחברו עם הסיסמה שהגדרתם.

## חיבור מייל אמיתי (מומלץ: Mailgun)
1. הרשמו ל-Mailgun וחברו את הדומיין שלכם (עם רשומות DNS שהם ייתנו לכם).
2. במסך ה-Routes של Mailgun, הגדירו route שמפנה כל מייל שמגיע לדומיין שלכם אל:
   `https://הדומיין-שלכם-בפועל/webhooks/inbound`
3. מלאו ב-`.env` את `MAILGUN_DOMAIN` ו-`MAILGUN_API_KEY`.
4. כתובות המייל לכל רשימה נבנות אוטומטית מה-slug שהגדרתם, לדוגמה:
   - `ask+parenting@yourdomain.com` - שאלה חדשה
   - `ads+parenting@yourdomain.com` - מודעה חדשה
   - `reply+123@yourdomain.com` - תגובה לשאלה מספר 123 (נוצר אוטומטית בקישור שבמייל)

## פריסה (deployment)
המערכת היא Node.js/Express רגילה עם קובץ SQLite - מתאימה מצוין לפריסה על
Railway, Render או VPS פשוט. ודאו שהתיקייה `data/` נשמרת בין דיפלוימנטים
(disk מתמשך), אחרת מסד הנתונים יימחק בכל עדכון.

## מבנה הפרויקט
```
src/
  db.js            - סכמת מסד הנתונים (SQLite)
  templates.js      - העיצוב האחיד של המייל (המותג "בית" אחד + גוון לכל רשימה)
  compiler.js        - מרכיב ושולח את הגיליון השבועי, כולל תזמון cron
  routes/
    admin.js          - פאנל ניהול (login, dashboard, תור אישור)
    inbound.js         - webhook לקליטת מיילים נכנסים
    public.js           - טופס מודעה ציבורי, הרשמה, הסרה
  views/               - תבניות EJS לפאנל ולטופס המודעה
```
